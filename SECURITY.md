# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.3.x   | Yes       |
| < 0.3   | No        |

## Reporting a Vulnerability

If you discover a security vulnerability in mustel itself (not in the code mustel scans), please **do not open a public issue.**

Instead, email us directly at **mustel.oss@gmail.com** with:

1. A description of the vulnerability.
2. Steps to reproduce it.
3. The version of mustel you are using.

## Scope

The following are **in scope**:
- Vulnerabilities in mustel's core code, engines, or MCP server
- YAML pattern injection or loader exploits
- Dependency vulnerabilities in mustel's own requirements

The following are **out of scope**:
- Bugs found *by* mustel in your code (that's mustel working as intended)
- Vulnerabilities in ruff, bandit, or pip-audit themselves (report those upstream)
